# 画像管理ガイド

このガイドでは、画像の差し替え・追加・管理方法を説明します。

---

## 📁 画像の保存場所

すべての画像は以下のフォルダに配置してください:

```
public/images/
```

---

## 🖼️ 画像の差し替え手順

### 方法1: 既存ファイルを上書き（簡単）

既存の画像を新しいものに差し替える場合:

1. 新しい画像を用意
2. **既存のファイル名と同じ名前**にリネーム
3. `public/images/` フォルダに配置（上書き）
4. 完了！（コードの変更不要）

**例:**
```
既存: public/images/Blog_00.png
↓
新しい画像を「Blog_00.png」にリネーム
↓
public/images/Blog_00.png に上書き保存
```

### 方法2: 新しいファイル名で追加

1. 新しい画像を `public/images/` に配置
2. `CONTENT.md` を開く
3. 該当箇所の画像パスを変更
4. `npm run update-content` を実行

**例:**
```markdown
# CONTENT.md の編集例

画像パス:
```
/images/新しい画像.png  ← ここを変更
```
```

## 🎨 画像の推奨仕様

### サイズとアスペクト比

| 種類 | 推奨サイズ | アスペクト比 | 備考 |
|-----|----------|------------|------|
| CHAPTERメイン画像 | 1200x800px | 3:2 | レスポンシブ対応 |
| NEW_POSTサムネイル | 800x600px | 4:3 | カード表示用 |
| ロゴ | 512x512px | 1:1 | 透過PNG推奨 |

### ファイル形式

- **写真**: JPG (最適化済み、100KB以下推奨)
- **イラスト/ロゴ**: PNG (透過対応)
- **アニメーション**: GIF (500KB以下推奨)
- **動画**: MP4 (5MB以下推奨)

### ネーミングルール

**推奨:**
- 英数字とハイフン・アンダースコアのみ
- 例: `chapter-01-main.png`, `post_thumbnail_01.jpg`

**非推奨:**
- スペースを含むファイル名
- 日本語のみのファイル名（現在のファイルは互換性のため残しています）

## 🔧 画像最適化のヒント

### 1. ファイルサイズを減らす

無料ツール:
- [TinyPNG](https://tinypng.com/) - PNG/JPG圧縮
- [Squoosh](https://squoosh.app/) - 画像最適化

### 2. WebP形式に変換（任意）

WebP形式は軽量で高画質です。

```bash
# 既存のPNGをWebPに変換する例
# ImageMagickがインストールされている場合
convert input.png output.webp
```

### 3. レスポンシブ画像

大きな画像は複数サイズを用意すると表示が高速化します:
- デスクトップ: 1200px幅
- タブレット: 768px幅
- モバイル: 375px幅

## ⚠️ よくあるトラブル

### 画像が表示されない

**チェック項目:**
1. ファイルは `public/images/` にあるか？
2. ファイル名は正確か？（スペース、大文字小文字に注意）
3. `CONTENT.md` のパスは `/images/` で始まっているか？

**正しいパス例:**
```
/images/Blog_00.png  ← 正解
images/Blog_00.png   ← 間違い（先頭の/が必要）
/public/images/Blog_00.png  ← 間違い（publicは不要）
```

### 画像が古いまま

**対処法:**
1. ブラウザのキャッシュをクリア
2. 開発サーバーを再起動
   ```bash
   # Ctrl+C で停止
   npm run dev
   ```

### ファイル名に日本語が含まれる

現在のファイルは互換性のために日本語名を許容していますが、今後は英数字での命名を推奨します。
